hexo 图片插入

关于 hexo 插入图片这个事情，搞的头都炸了，有很多种方式来插入图片，不过使用了那个 hexo-asset-image 插件之后真的各种问题，所以没有使用插件，对于 Markdown 插入图片的两种方式也没有使用，使用了 hexo 官方推荐的标签方式进行了图片的插入，不过又带来一个问题那就是本地图片越来越多，导致空间不够，同时访问速度严重的变慢，真的很让人烦恼，难道要使用图床吗，目前对于图床还是很了解，截止写这个的时候还没有进行尝试的打算。下面附上官方的插入图片的方式。 {% asset_img name.jpg title %}

使用这种方式可以在首页同时显示，截止写文章时，并没有发现这种方式可以在首页同时显示图片可能是自己设置的问题，猜测是配置文件中的首页内容预览的问题。先放着，到处都是坑。

我写博客好随意啊😂

SEO 优化

自己进行的优化还是有很多问题的。 谷歌的优化目前搞定了 http://windmill.site 。 但是 http://www.windmill.site 还没有搞定（问题和百度的一样），还有就是提交的抓取内容不多，导致搜索的范围较少。 至于百度的始终不能发现我的 robots.txt，我心中充满了困惑，不过想着自己的博客写得这么随意，也没有什么技术指导性，也是参考别人的博客进行的优化，所以更像是简单的记录。因此能不能被别人搜索到也没啥关系自己玩的很开心。

SEO 优化这个想起来的时候再填坑吧😂

以后博客的发展方向

还没想好自己的这个博客的发展方向，目前更多是兴趣使然，写着开心的。所以这又带来一个巨大的坑，我自己的日记是写在 Bear 上面的，使用体验目前来说还是 OK 的，就是主题界面有点审美疲劳了。今天用 atom 写博客，感觉舒服很多，看来是 Markdown-preview-enhanced 插件带来的良好体验，点击 .md 文件自动跳出来预览，不用手动打开，同时网址链接自动生成链接，不用使用 Markdown 的方法，总体来说体验很好。

随意的坑，随意的填，随意的我，随意的玩。😂
